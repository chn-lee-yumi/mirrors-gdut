Docker镜像使用帮助

使用说明

基于 Harbor 实现的容器镜像代理仓库，支持搜索已经缓存下来的镜像。

1. 对于使用 Docker-Hub 镜像仓库

docker pull registry.gdut.edu.cn/docker/google/pause:latest
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/docker/library/nginx:latest

2. 对于使用阿里云 Docker 镜像仓库

docker pull registry.gdut.edu.cn/aliyun/portainer/portainer-ce:latest
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/aliyun/library/java:latest
由于阿里云代理仓库的时效性问题,这里不建议在此使用latest标签拉取镜像

3. 对于使用 GitHub Container Registry (ghcr.io) 镜像仓库

docker pull registry.gdut.edu.cn/ghcr.io/hay-kot/homebox:latest
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/ghcr.io/library/test:latest

4. 对于使用 RedHat Quay.io (quay.io) 镜像仓库

docker pull registry.gdut.edu.cn/quay.io/coreos/flannel:v0.15.1
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/quay.io/library/test:latest

5. 对于使用 Microsoft Artifact Registry (mcr.microsoft.com) 镜像仓库

docker pull registry.gdut.edu.cn/mcr.microsoft.com/coreos/flannel:v0.15.1
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/mcr.microsoft.com/library/test:latest

6. 对于使用 Google Container Registry (gcr.io) 镜像仓库

docker pull registry.gdut.edu.cn/gcr.io/kaniko-project/executor:v1.13.0-debug
# 如果是官方镜像或者单级存储则加上默认的库名称library即可
docekr pull registry.gdut.edu.cn/gcr.io/library/test:latest

7. 对于使用 k8s.gcr.io 镜像仓库

docker pull registry.gdut.edu.cn/k8s.gcr.io/kube-proxy:v1.24.16

拉下来后该重新打Tag就重新打Tag
